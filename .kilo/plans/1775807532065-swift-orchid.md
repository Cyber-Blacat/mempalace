## Analysis Summary

Based on codebase exploration, I've identified:

### Project Overview
- **MemPalace**: AI memory system for projects and conversations
- **Language**: Python 3.9+
- **Package Manager**: Hatchling build system
- **Key Dependencies**: ChromaDB, PyYAML, rank-bm25
- **Development Dependencies**: pytest, pytest-cov, ruff, psutil

### Testing Infrastructure
- **Test Framework**: pytest with comprehensive fixtures in `tests/conftest.py`
- **Test Markers**: `@pytest.mark.benchmark`, `@pytest.mark.slow`, `@pytest.mark.stress`
- **Default Test Exclusions**: Benchmarks, slow, and stress tests excluded by default
- **Coverage**: Minimum 30% required
- **Test Isolation**: HOME env redirected to temp directory for isolation

### Linting & Formatting
- **Linter**: Ruff with E, F, W codes selected (E501 ignored)
- **Formatter**: Ruff format with double quotes
- **Line Length**: 100 characters
- **Pre-commit hooks**: Configured for automatic linting/formatting

### CI/CD Pipeline
- **GitHub Actions**: Tests on Python 3.9, 3.11, 3.13
- **Linting Job**: Separate from tests
- **Dependency Installation**: `pip install -e ".[dev]"`

### Code Style Patterns
1. **Naming Conventions**:
   - Functions/Variables: `snake_case`
   - Classes: `PascalCase`
   - Constants: `UPPER_SNAKE_CASE`

2. **Import Organization**:
   - Standard library imports first
   - Third-party imports next
   - Local imports last
   - Absolute imports preferred: `from .config import MempalaceConfig`

3. **Type Hints**:
   - Consistent usage on public functions/methods
   - Return type annotations common
   - From `typing` module: `List`, `Dict`, `Optional`, etc.

4. **Error Handling**:
   - Custom exception classes (e.g., `SearchError`)
   - Exception chaining with `raise ... from e`
   - User-friendly error messages
   - Graceful degradation where possible

5. **Docstrings**:
   - Module-level docstrings describing purpose
   - Class docstrings with attributes and examples
   - Function docstrings with Args, Returns, Raises sections

6. **File Structure**:
   - Core package: `mempalace/`
   - Tests: `tests/`
   - Benchmarks: `benchmarks/`
   - Examples: `examples/`
   - Hooks: `hooks/`
